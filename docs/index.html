Introduction

pcUtils est un framework modulaire pour la création d'interfaces utilisateur en terminal. Il fournit une architecture extensible basée sur les composants et les plugins, permettant de créer rapidement des applications interactives et maintenables.

Key Features

  • Modern and responsive terminal UI components
  • Plugin-based validation system with auto-discovery
  • Extensive theming and customization options

Architecture

L'architecture de pcUtils est organisée en trois parties principales :

  • Composants UI : Widgets réutilisables pour l'interface utilisateur
  • Système de Validation : Validation des données avec plugins auto-découverts
  • Système de Plugins : Extensions modulaires pour ajouter des fonctionnalités

Requirements

  • Python 3.7+
  • textual>=0.1.18
  • email-validator>=2.0.0

Créer un Composant

Structure d'un Composant

from textual.app import ComposeResult
from textual.widgets import Input
from ..base import BaseField

class MonComposant(BaseField):
    """Description du composant"""
    
    def compose(self) -> ComposeResult:
        """Création de l'interface"""
        yield from super().compose()
        
        # Créer les widgets
        mon_input = Input(
            id=f"input_{self.field_id}",
            value=str(self.value) if self.value is not None else "",
            placeholder=self.field_config.get('placeholder', '')
        )
        
        yield mon_input
        
    def on_input_changed(self, event: Input.Changed) -> None:
        """Gestion des changements"""
        if event.input.id == f"input_{self.field_id}":
            self._update_dependent_fields(event.input.value)
            
    def get_value(self) -> str:
        """Récupération de la valeur"""
        input_widget = self.query_one(f"#input_{self.field_id}", Input)
        return input_widget.value if input_widget else ""

Guide de Configuration YAML

Structure du Fichier YAML

Le fichier YAML d'un plugin se compose de deux parties :

1. Métadonnées du Plugin
# En-tête du plugin
name: Nom du Plugin           # Nom affiché dans l'interface
description: Description     # Description détaillée
version: 1.0                 # Version du plugin
category: Système           # Catégorie (Système, Réseau, etc.)
multiple: true               # Peut être exécuté plusieurs fois
icon: 💻                   # Icône du plugin
2. Configuration des Champs

Définition des champs de configuration sous config_fields :

nom_du_champ:
  type: text                    # Type du champ (text, number, select, ip, etc.)
  label: "Mon Champ"            # Label affiché
  description: "Description"     # Aide contextuelle
  placeholder: "Exemple"        # Texte d'exemple
  required: true                # Champ obligatoire
  default: "Valeur"             # Valeur par défaut
  
  # Validation (cumulative)
  validate:
    - not_empty                 # Champ non vide
    - number                    # Doit être un nombre
    - max_length: 12            # Longueur maximale
    - min: 0                    # Valeur minimale
    - max: 100                  # Valeur maximale
    - pattern: "^[A-Za-z0-9_]+$" # Expression régulière
    
  # Valeurs dynamiques (pour select)
  dynamic_options:
    script: mon_script.py
    function: get_options
Types de Champs
Basiques
  • text - Texte simple
  • number - Nombre
  • password - Mot de passe
  • textarea - Texte multi-lignes
Sélection
  • select - Liste déroulante
  • checkbox - Case à cocher
  • checkbox_group - Groupe de cases
  • radio - Boutons radio
Spéciaux
  • ip - Adresse IP
  • date - Date
  • file - Fichier
  • directory - Dossier
Avancés
  • json - Données JSON
  • code - Éditeur de code
  • color - Sélecteur de couleur
  • table - Tableau de données
Validations Disponibles
  • not_empty - Non vide
  • number - Nombre
  • min/max - Limites numériques
  • length - Longueur texte
  • pattern - Expression régulière
